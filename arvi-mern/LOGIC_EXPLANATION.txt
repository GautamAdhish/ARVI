PROJECT OVERVIEW
================================================================================
ARVI is converted from a static HTML website to a full MERN (MongoDB, Express, 
React, Node.js) stack application. Currently, only the React frontend is 
implemented. Backend will be added later.

TECHNOLOGY STACK
================================================================================
Frontend:
- React 18.2.0 - UI Library for building components
- React DOM 18.2.0 - Rendering React components to DOM
- React Router DOM 6.15.0 - Client-side routing (prepared for future use)
- Axios 1.5.0 - HTTP client for API calls (ready for backend integration)
- React Scripts 5.0.1 - Build and dev tools

Development:
- Node.js - JavaScript runtime
- npm - Package manager

KEY COMPONENTS & LOGIC
================================================================================

1. HEADER COMPONENT (Header.js)
   Purpose: Navigation bar with smooth scrolling
   Logic:
   - Displays ARVI logo on left
   - Navigation links: Products, Blog, About Us, More
   - scrollToSection() function smoothly scrolls to section IDs
   - Uses anchor links (#products, #about, #contact) for navigation
   - Sticky positioning keeps header visible during scroll

2. HERO COMPONENT (Hero.js)
   Purpose: Landing section with call-to-action buttons
   Logic:
   - Displays hero image as background
   - Contains main heading and tagline
   - Two buttons: "Join Waitlist" and "View Products"
   - Button handlers show alerts (will be replaced with actual functionality later)
   - Uses button-group layout for responsive button arrangement

3. PRODUCTS COMPONENT (Products.js)
   Purpose: Display product catalog
   Logic:
   - Uses state array of product objects with id, title, description, image
   - .map() function iterates through products array
   - Renders dynamic product cards with images and descriptions
   - Images use unsplash CDN for placeholder images
   - 2-column grid layout (responsive - 1 column on mobile)

4. BENEFITS COMPONENT (Benefits.js)
   Purpose: Display why customers should choose ARVI
   Logic:
   - Benefits array contains 6 benefit objects
   - .map() function renders each benefit as a card
   - Each card shows title and description
   - 2-column grid layout (responsive to 1-column on mobile)
   - Cards have left border accent color

5. LIFESTYLE COMPONENT (Lifestyle.js)
   Purpose: Showcase lifestyle imagery and messaging
   Logic:
   - 2-column layout: image on left, text on right
   - Image uses Unsplash URL with background-image CSS
   - Text section contains heading and descriptive paragraphs
   - Responsive design: stacks vertically on mobile

6. MATERIALS COMPONENT (Materials.js)
   Purpose: Display material safety information
   Logic:
   - Materials array contains 4 material types
   - .map() function creates grid items for each material
   - Auto-fit CSS Grid adjusts columns based on screen size
   - Emphasizes safety and quality

7. COLOUR COMPONENT (Colour.js)
   Purpose: Display available color options
   Logic:
   - Shows current launch color (Sage Green)
   - Emoji placeholder (🎨) instead of color swatch
   - Additional text: "More colours coming soon"
   - Centered layout

8. ABOUT COMPONENT (About.js)
   Purpose: Company information section
   Logic:
   - Single section describing ARVI's mission
   - Emphasizes thoughtful design and practical use
   - ID="about" allows navigation from header

9. NEWSLETTER COMPONENT (Newsletter.js)
   Purpose: Email subscription form
   Logic:
   - useState() hook manages email state and submitted status
   - Email input validates and updates state on change
   - handleSubmit() prevents default form behavior
   - Shows success message for 3 seconds after submission
   - Future: Will integrate with backend API
   - Input focuses on accessibility and user experience

10. CONTACT COMPONENT (Contact.js)
    Purpose: Display contact information
    Logic:
    - Links to email, Instagram, and WhatsApp
    - Uses target="_blank" to open links in new tabs
    - rel="noopener noreferrer" security for external links
    - Email link uses mailto: protocol
    - WhatsApp link uses wa.me: protocol
    - ID="contact" enables navigation from header

11. FOOTER COMPONENT (Footer.js)
    Purpose: Footer with links and copyright
    Logic:
    - Links to Privacy Policy and Contact
    - Copyright notice with year 2026
    - Consistent styling with header

12. APP COMPONENT (App.js)
    Purpose: Main component that orchestrates all sections
    Logic:
    - Imports all components
    - Imports global CSS styles
    - Renders components in logical order
    - Creates single-page application structure
    - No page reload required when navigating

STYLING APPROACH
================================================================================
CSS Organization:
- Single global CSS file (src/styles/index.css)
- BEM-like naming convention (e.g., .product-grid, .product-card)
- Mobile-first responsive design with @media queries
- Breakpoint: 768px for tablet/mobile view

Color Scheme:
- Primary: #2c3e50 (dark blue-gray)
- Accent: #7cb342 (sage green)
- Light: #f9f8f6 (off-white)
- Text: #333, #666 (various grays)

Responsive Design:
- Header navigation: 30px gap → 15px gap on mobile
- Button group: flex row → flex column on mobile
- Product grid: 2 columns → 1 column on mobile
- Benefits grid: 2 columns → 1 column on mobile

STATE MANAGEMENT
================================================================================
Currently using React Hooks:
- useState() in Newsletter.js for email input and submission status
- Could expand to useContext for global state (themes, user data, etc.)
- Could add Redux for complex state management in future

FUTURE ENHANCEMENTS - BACKEND INTEGRATION
================================================================================
1. Database (MongoDB):
   - Users collection: email, name, registration date
   - Products collection: product data with pricing
   - Orders collection: purchase history

   - Implement React Router for page navigation
   - Add authentication for user accounts
   - Implement shopping cart functionality
   - Add payment gateway integration (Stripe/PayPal)

RUNNING THE PROJECT
================================================================================
Prerequisites:
- Node.js (v14+) and npm installed

Installation:
1. Navigate to project: cd arvi-mern
2. Install dependencies: npm install
3. Start dev server: npm start
4. Open browser to http://localhost:3000

Building for Production:
npm run build
- Creates optimized build in /build folder
- Ready for deployment
